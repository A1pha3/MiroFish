## 高级主题概览

### 1. GraphRAG 深度解析

MiroFish 使用 Zep Cloud 的 GraphRAG 能力进行知识提取和检索增强生成：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    GraphRAG 核心流程                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  输入文档                                                           │
│    │                                                                │
│    ▼                                                                │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  1. 文本分块 (Chunking)                                       │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │ "这是一段长文本..." + 重叠 → [Chunk1][Chunk2][Chunk3] │  │  │
│  │  │                                                          │  │  │
│  │  │ 参数: chunk_size=500, chunk_overlap=50                   │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│    │                                                                │
│    ▼                                                                │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  2. 本体生成 (Ontology Generation)                             │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │ LLM 分析 → 识别实体类型 → 识别关系类型                 │  │  │
│  │  │                                                          │  │  │
│  │  │ 输出: {                                                  │  │  │
│  │  │   entity_types: [Person, Organization, ...]            │  │  │
│  │  │   edge_types: [KNOWS, WORKS_FOR, ...]                   │  │  │
│  │  │ }                                                       │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│    │                                                                │
│    ▼                                                                │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  3. 实体和关系提取                                             │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │ 对每个文本块:                                           │  │  │
│  │  │   LLM 提取 → 实体: {name, type, description}          │  │  │
│  │  │            → 关系: {source, target, type, weight}     │  │  │
│  │  │                                                          │  │  │
│  │  │ 跨块关联 → 合并相同实体 → 去重 → 构建图谱           │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│    │                                                                │
│    ▼                                                                │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  4. 图谱构建和存储                                             │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │ Zep Cloud 存储:                                        │  │  │
│  │  │   · Nodes (实体) - 带属性和 embedding                │  │  │
│  │  │   · Edges (关系) - 带类型和权重                       │  │  │
│  │  │   · Communities (社群) - 自动聚类                     │  │  │
│  │  │                                                          │  │  │
│  │  │ 支持的操作:                                             │  │  │
│  │  │   · InsightForge - 深度洞察检索                        │  │  │
│  │  │   · PanoramaSearch - 全景搜索                          │  │  │
│  │  │   · EntityLookup - 实体查询                            │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│    │                                                                │
│    ▼                                                                │
│  可查询的知识图谱                                                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 2. OASIS 智能体架构

OASIS (Open Agent Social Interaction Simulation) 是一个专业的社交模拟引擎：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OASIS 智能体层次架构                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  1. Agent Profile 层 (智能体人设)                             │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │ {                                                       │  │  │
│  │  │   user_id: 1,                                           │  │  │
│  │  │   screen_name: "jiabaoyu_clan",                         │  │  │
│  │  │   name: "贾宝玉",                                       │  │  │
│  │  │   personality: "温和、多情、反叛传统",                    │  │  │
│  │  │   bio: "荣国府公子...",                                 │  │  │
│  │  │   followers_count: 245,                                │  │  │
│  │  │   ...                                                   │  │  │
│  │  │ }                                                       │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                          │                                        │
│                          ▼                                        │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  2. Memory 层 (记忆系统)                                      │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │ 短期记忆 (STM):                                         │  │  │
│  │  │   · 最近看到的帖子                                      │  │  │
│  │  │   · 最近的互动                                         │  │  │
│  │  │                                                         │  │  │
│  │  │ 长期记忆 (LTM):                                         │  │  │
│  │  │   · Zep 图谱存储                                       │  │  │
│  │  │   · 人际关系网络                                       │  │  │
│  │  │   · 过去的重要事件                                     │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                          │                                        │
│                          ▼                                        │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  3. Decision 层 (决策机制)                                    │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │ 每轮决策流程:                                          │  │  │
│  │  │   1. 观察环境 (Observe)                                  │  │  │
│  │  │      └─> 查看新帖子、检查状态                          │  │  │
│  │  │   2. 检索记忆 (Retrieve)                                │  │  │
│  │  │      └─> 从 Zep 获取相关信息                            │  │  │
│  │  │   3. 形成意图 (Intent)                                  │  │  │
│  │  │      └─> LLM 生成行动意图                               │  │  │
│  │  │   4. 选择动作 (Act)                                     │  │  │
│  │  │      └─> 执行或跳过                                      │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                          │                                        │
│                          ▼                                        │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  4. Action 层 (动作执行)                                      │  │
│  │  ┌─────────────────────────────────────────────────────────┐  │  │
│  │  │ Twitter Actions:                                        │  │  │
│  │  │   · CREATE_POST - 发布推文                              │  │  │
│  │  │   · LIKE_POST - 点赞                                    │  │  │
│  │  │   · RETWEET - 转发                                      │  │  │
│  │  │                                                         │  │  │
│  │  │ Reddit Actions:                                         │  │  │
│  │  │   · CREATE_POST - 发布帖子                              │  │  │
│  │  │   · CREATE_COMMENT - 发表评论                          │  │  │
│  │  │   · UPVOTE - 投票                                        │  │  │
│  │  └─────────────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────────────┘  │
```

### 4. IPC 通信机制

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IPC 进程间通信架构                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Flask 主进程                                              OASIS 子进程
│  ┌─────────────────┐                                      ┌─────────────────┐
│  │ Flask Backend   │                                      │ OASIS Engine    │
│  │                 │                                      │                 │
│  │ ┌─────────────┐ │                                      │ ┌─────────────┐ │
│  │ │ API Handler │ │                                      │ │ Simulation  │ │
│  │ └──────┬──────┘ │                                      │ │ Loop        │ │
│  │        │        │                                      │ └──────┬──────┘ │
│  └────────┼────────┘                                      └────────┼────────┘
│           │                                                        │
│           │ 文件系统                                               │
│           ▼                                                        ▼
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  IPC 目录: backend/uploads/simulations/{sim_id}/              │  │
│  │                                                              │  │
│  │  ┌─────────────────────────────────────────────────────────┐ │  │
│  │  │ ipc_commands/         ← Flask 写入命令                  │ │  │
│  │  │   ├── interview_request_001.json                         │ │  │
│  │  │   └── batch_interview_001.json                            │ │  │
│  │  │                                                         │ │  │
│  │  │ ipc_responses/        ← OASIS 写入响应                  │ │  │
│  │  │   ├── interview_response_001.json                        │ │  │
│  │  │   └── batch_interview_response_001.json                   │ │  │
│  │  │                                                         │ │  │
│  │  │ status.json           ← 双向读写状态                     │ │  │
│  │  └─────────────────────────────────────────────────────────┘ │  │
│  └───────────────────────────────────────────────────────────────┘  │
│           │                                                        │
│           │ 轮询                                                   │
│           ▼                                                        │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  SimulationIPCServer                                        │  │
│  │  ┌─────────────────────────────────────────────────────────┐ │  │
│  │  │ · poll_commands()    - 轮询新命令 (Flask 调用)         │ │  │
│  │  │ · send_response()    - 发送响应 (OASIS 调用)           │ │  │
│  │  │ · update_status()    - 更新状态 (双向)                 │ │  │
│  │  └─────────────────────────────────────────────────────────┘ │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```
