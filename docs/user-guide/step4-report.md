## 核心概念：Report Agent 工作原理

### ReACT 模式详解

Report Agent 使用 **ReACT (Reasoning + Acting + Observation)** 循环模式：

```
┌─────────────────────────────────────────────────────────────┐
│                     ReACT 思考-行动循环                      │

│  │ "用户想知道贾宝玉和林黛玉的关系走向。我需要:        │   │

│  │  2. 分析模拟过程中的互动变化                       │   │
│  │  3. 识别影响关系的关键事件                         │   │

│  ┌────────────────────────────────────────────────────┐     │

#### 需求模板

| 分析类型 | 需求模板 |
|---------|----------|
| **关系分析** | "分析 X 和 Y 的关系走向，考虑 Z 的影响" |
| **事件预测** | "预测 X 事件的可能结果，分析概率和因素" |
| **群体动态** | "分析群体 X 的互动模式，识别关键人物和冲突" |
| **演化趋势** | "分析系统从第 A 轮到第 B 轮的演化趋势" |
| **综合分析** | "全面分析模拟结果，给出预测结论和建议" |

#### 高级参数

| 参数 | 说明 | 默认值 | 调整建议 |
|------|------|--------|----------|
| **最大工具调用** | 最多调用工具次数 | 5 | 复杂分析→7-10 |
| **反思轮数** | 自我反思轮数 | 2 | 需要深度→3-4 |
| **温度参数** | 生成随机性 | 0.5 | 创意→0.7-0.9，精确→0.3-0.5 |

```

### 练习 4.1：报告质量评估

| 维度 | 评估标准 | 你的判断 |
|------|----------|----------|
| 数据引用 | 是否引用具体轮次和动作 | |
| 逻辑连贯 | 分析是否有逻辑支撑 | |
| 结论明确 | 预测结论是否清晰 | |
| 多角度 | 是否考虑多种可能性 | |

### 练习 4.2：需求优化

将以下需求优化为高质量需求：

```
原需求: "分析一下关系"

优化后: "分析贾宝玉和林黛玉在第 20-40 轮期间的关系变化，
        识别导致变化的关键事件，并评估这些事件对最终结局的影响"
```

### 练习 4.3：追问技巧

根据报告内容设计有效追问：

```
报告结论: "关系可能恶化"

追问1: "具体哪些事件导致恶化？" (深入细节)
追问2: "有防止恶化的可能吗？" (探索假设)
追问3: "恶化是否必然，还是概率事件？" (验证结论)
```
