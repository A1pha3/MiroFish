## 最佳实践

### 数据准备

```
✅ 好的材料特征:
   · 结构清晰，有明确的人物和事件
   · 包含丰富的关系描述
   · 时间线明确
   · 文本连贯完整

❌ 不好的材料:
   · 纯数据列表
   · 过于碎片化的内容
   · 缺乏上下文
   · 时间线混乱
```

### 模拟配置

```
配置原则:
1. 从小规模开始
   - 首次测试: 10-20 轮，10-30 个智能体
   - 正式预测: 40 轮，30-50 个智能体
   - 深度分析: 80+ 轮，100+ 个智能体

2. 选择合适的平台
   - 关注即时反应 → Twitter
   - 关注深度讨论 → Reddit
   - 全面分析 → 双平台

3. 明确预测需求
   - 具体说明分析对象
   - 明确分析维度
   - 指定期望的输出形式
```

### 报告解读

```
解读要点:
1. 关注数据支撑
   - 报告结论是否有模拟数据支持
   - 引用的具体事件和轮次

2. 理解概率性质
   - 预测结论通常是概率性的
   - 关注可能性而非确定性

3. 结合交互验证
   - 使用访谈功能验证结论
   - 对比不同智能体的观点
```

## 常见问题

### Q: 图谱构建失败怎么办？

**排查步骤**:
1. 检查文件格式（仅支持 PDF/MD/TXT）
2. 检查文件大小（不超过 50MB）
3. 检查 LLM API 配置
4. 检查 Zep API Key 是否有效
5. 查看详细错误日志

### Q: 模拟运行时间很长？

**影响因素**:
- 智能体数量（越多越慢）
- 模拟轮数（线性增长）
- LLM 响应速度（取决于模型）
- 网络延迟

**建议**: 首次使用选择 10-20 轮测试

### Q: 报告质量不满意？

**改进方法**:
1. 优化预测需求描述（更具体）
2. 增加模拟轮数（更多互动数据）
3. 调整高级参数（温度、工具调用次数）
4. 使用对话功能深入追问

### Q: 智能体回答不相关？

**可能原因**:
- 问题不够明确
- Profile 质量不高
- 需要调整提问方式

**改进**: 使用更具体、开放式的问题
