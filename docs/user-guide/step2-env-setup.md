# Step 2: 环境搭建

> 配置模拟环境，将知识图谱转化为可运行的智能体系统

**难度**: ⭐⭐ | **预计时间**: 15 分钟 | **前置知识**: Step 1 完成

---

## 学习目标

完成本步骤后，你将能够：

### 基础目标（必掌握）

- [ ] 理解模拟环境在预测流程中的作用
- [ ] 查看和分析图谱中的实体信息
- [ ] 选择合适的实体类型和数量
- [ ] 理解智能体 Profile 的构成
- [ ] 配置和准备模拟环境

### 进阶目标（建议掌握）

- [ ] 评估不同实体选择对模拟效果的影响
- [ ] 理解 Profile 参数对智能体行为的作用
- [ ] 根据预测需求调整模拟配置

### 专家目标（挑战）

- [ ] 设计最优的智能体组合策略
- [ ] 预测并控制模拟的计算成本

#### 实体数量对照

| 实体数量 | 预计模拟时间 | 内存占用 | 推荐场景 |
|---------|-------------|---------|----------|
| **10-30** | 5-15 分钟 | ~500MB | 快速测试、概念验证 |
| **30-50** | 15-30 分钟 | ~1GB | 标准预测任务 |
| **50-100** | 30-60 分钟 | ~2GB | 复杂场景、全景分析 |
| **100+** | 1小时+ | 4GB+ | 大规模研究 |

#### Profile 生成选项对比

| 选项 | 生成方式 | 质量 | 速度 | API 消耗 | 推荐场景 |
|------|---------|------|------|----------|----------|
| **使用 LLM** | 调用 LLM 分析实体生成 | ⭐⭐⭐⭐⭐ | 慢 | 高 | 正式预测、需要高质量人设 |
| **不使用 LLM** | 基于实体描述生成 | ⭐⭐⭐ | 快 | 无 | 快速测试、原型验证 |

#### 配置参数说明

| 参数 | 范围 | 默认值 | 影响 | 调整建议 |
|------|------|--------|------|----------|
| **模拟轮数** | 10-100 | 40 | 轮数多→关系变化更明显 | 快速测试→10-20，完整分析→40+ |
| **最小活跃度** | 0-1 | 0.3 | 控制最少互动 | 人物少→可提高 |
| **最大活跃度** | 0-1 | 0.8 | 控制最多互动 | 避免过于活跃 |
| **初始状态** | - | 中性 | 影响初始情绪 | 根据故事背景设定 |

## 故障排查

### 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| **Profile 质量差** | LLM 模型太弱 | 切换到更好的模型 |
| **实体太多** | 筛选条件太宽 | 减少实体类型选择 |
| **生成很慢** | 并行数太低 | 增加并行生成数 |
| **配置不合理** | 自动生成问题 | 手动调整高级配置 |

### 练习 2.1：实体数量评估

**任务**：给定 120 个实体的图谱，选择多少实体合适？

| 场景 | 推荐数量 | 理由 |
|------|---------|------|

| 全景分析 | 80-100 | 覆盖主要实体 |

### 练习 2.2：预测需求分析

**任务**：为以下预测需求分析配置

```
需求: "预测公司合并后，两个团队的文化冲突"
```

**分析**：
- 关键实体：两个团队的核心成员
- 模拟轮数：20-30 轮（文化冲突显现较快）
- 平台：Reddit（适合深度讨论）
- 初始状态：警惕/防御
